Proyectos seleccionados

Algunos de los proyectos que muestran cómo conecto firmware, comunicaciones y backend.

STM32WL LoRaWAN End-Node

Firmware · IoT · LoRaWAN

Desarrollo de firmware para un nodo LoRaWAN basado en STM32WL, incluyendo drivers para sensores, manejo de bajo consumo, y comunicación segura con un network server.

  • Rol: Diseño de arquitectura, implementación de firmware, validación y pruebas integrales.
  • Tecnologías: C, STM32Cube, LoRaWAN stack, SPI/I²C, UART, Git.
  • Highlights: optimización de consumo, manejo de reconexión, integración con gateway y backend MQTT.

Medidor de calidad eléctrica con ADE9000/STPM32

Firmware · Drivers · Energía

Implementación de drivers para medidores de energía y calidad eléctrica, lectura de parámetros, filtros y procesamiento básico para integrar con sistemas de monitoreo.

  • Rol: Desarrollo de drivers, pruebas de laboratorio y soporte a validación en campo.
  • Tecnologías: C, SPI, MCU STM32, ADE9000/STPM32, herramientas de medición.
  • Highlights: robustez ante ruido eléctrico, calibración y pruebas de producción.

Controlador UDP para simulador de vuelo

Networking · C · Herramientas

Librería en C para controlar un simulador de vuelo mediante tramas UDP, integrando la documentación del fabricante y análisis de tráfico con Wireshark.

  • Rol: Diseño del protocolo, implementación del cliente UDP y pruebas end-to-end.
  • Tecnologías: C, sockets UDP, Wireshark, Linux.
  • Highlights: análisis de tramas, tolerancia a errores en red, documentación técnica.

Backend & Dashboard para sensores IoT

Full-Stack · MQTT · REST

Backend ligero para recibir datos de sensores vía MQTT/HTTP, almacenarlos y visualizarlos en un dashboard simple. Incluye herramientas de calibración e inspección de datos.

  • Rol: Diseño de API, integración con broker, desarrollo de vistas y flujos de calibración.
  • Tecnologías: Node.js / Python, MQTT, REST, HTML/CSS/JS o Flutter.
  • Highlights: flujo completo dispositivo → broker → backend → dashboard.

Herramientas de prueba y servicios en Linux

Linux · Scripting · Automatización

Scripts y servicios en Linux para automatizar pruebas, recolectar datos de dispositivos y facilitar calibraciones y diagnósticos en producción.

  • Rol: Diseño de scripts, integración con dispositivos y documentación para el equipo.
  • Tecnologías: Bash, Python, systemd, herramientas de red.
  • Highlights: reducción de tiempo de pruebas y mejora en trazabilidad de fallas.
